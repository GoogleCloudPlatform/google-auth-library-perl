# UPB Build Integration

The Perl `Protobuf` module directly embeds the `upb` library source code, compiling it as part of the XS extension build process using `ExtUtils::MakeMaker`.

## Strategy: Static Library Compilation

-   **Bazel Prerequisite:** Before running the Perl build, specific UPB C sources must be generated by Bazel. This is done by running `bazel build //src/google/protobuf:descriptor_upb_proto` from the repository root.
-   **Source Files:** The `Makefile.PL` uses `File::Find` to gather necessary `.c` files from `../upb/` (excluding conformance and stage0). It also includes `descriptor.pb.cc` from `../src/` and the Bazel-generated `descriptor.upb.c` and `descriptor.upb_minitable.c`. Helper C files from `xs/` are also included.
-   **Include Paths:** The `INC` parameter in `WriteMakefile` and the compiler flags in `MY::postamble` include necessary paths for headers, including `.` for `xs/`, `../upb`, `../src`, and the Bazel-generated directories.
-   **Object Files:** Custom rules in `MY::postamble` compile each C/C++ source file into individual object files within the `perl/` directory.
-   **Static Library:** The generated object files are archived into `libprotobuf_common.a` within `MY::postamble`.
-   **`-fPIC`:** Ensured for all compiled code.
-   **Linking:** Each Perl XS module `.so` file (built via `XSMULTI`) links against `libprotobuf_common.a` using the `LIBS` option in `WriteMakefile`.
-   **`XSMULTI => 1`:** This option is used to build separate shared libraries for each module in `lib/Protobuf/`.

## Advantages

-   **Bundling:** `upb` is self-contained.
-   **Version Control:** `upb` version is tied to the repo.
-   **Simplified User Build:** Standard Perl toolchain.

## Disadvantages

-   **Slower Compilation:** Compiling all UPB sources.

## Alternatives Considered

-   **Amalgamation:** Difficult to manage flags.
-   **Linking all objects into each .so:** Larger `.so` files.
-   **System libupb:** External dependency for users.

## Conclusion

Compiling UPB sources into a static library and linking the `XSMULTI`-generated XS modules against it is the current approach.
